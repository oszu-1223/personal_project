# 네트워크 취약점 탐색 프로그램


## 프로젝트 소개
* 네트워크 서버의 취약점을 탐색하는 과정을 게임 형태로 구현한 프로그램
* 사용자가 웹 서버, 로그인 서버, 데이터베이스 서버 중 하나를 선택하여 각 서버의 보안 항목을 검사하고, 발견된 취약점에 따라 위험 점수를 산정함

### UI 구성
<img width="331" height="472" alt="image" src="https://github.com/user-attachments/assets/ef9a0459-8503-494a-bef7-ecea3afc0e8c" />


1. 웹 서버 화면 (HTTPS 사용 여부 검사 / 입력값 검증 검사 / 파일 업로드 검사 중 하나를 선택하는 화면 출력)
2. 로그인 서버 화면 (비밀번호 검사 / 로그인 제한 검사 / 2차 인증 검사 중 하나를 선택하는 화면 출력)
3. 데이터베이스 서버 화면 (접근 권한 검사 / 데이터 암호화 검사 / 백업 보안 검사 중 하나를 선택하는 화면 출력)

### 함수 설계
* web :  웹 서버의 보안 상태를 검사하는 함수
  * HTTPS 사용 여부, 입력값 검증, 파일 업로드 보안 항목을 선택하여 검사하고 취약점 발견 여부에 따라 위험 점수를 증가시킴
  * 1,2,3 외의 숫자를 입력했을 시 다시 입력하라는 문구를 출력함
<img width="348" height="332" alt="image" src="https://github.com/user-attachments/assets/4bcf0339-551c-4637-9d45-2d87b5bdd2ae" />

* login : 로그인 서버의 보안 상태를 검사하는 함수
  * 비밀번호, 로그인 시도 제한, 2차 인증 항목을 선택하여 검사하고 취약점 발견 여부에 따라 위험 점수를 증가시킴
  * 1,2,3 외의 숫자를 입력했을 시 다시 입력하라는 문구를 출력함
<img width="372" height="291" alt="image" src="https://github.com/user-attachments/assets/02757332-41f0-46e8-b533-c17c1fa600ad" />

* database : 데이터베이스 서버의 보안 상태를 검사하는 함수
  * 접근 권한, 데이터 암호화, 백업 보안 항목을 선택하여 검사하고 취약점 발견 여부에 따라 위험 점수를 증가시킴
  * 1,2,3 외의 숫자를 입력했을 시 다시 입력하라는 문구를 출력함
