# 네트워크 취약점 탐색 프로그램


## 프로젝트 소개
* 네트워크 서버의 취약점을 탐색하는 과정을 게임 형태로 구현한 프로그램
* 사용자가 웹 서버, 로그인 서버, 데이터베이스 서버 중 하나를 선택하여 각 서버의 보안 항목을 검사하고, 발견된 취약점에 따라 위험 점수를 산정함

### UI 구성

1. 웹 서버 화면 (HTTPS 사용 여부 검사 / 입력값 검증 검사 / 파일 업로드 검사 중 하나를 선택하는 화면 출력)
2. 로그인 서버 화면 (비밀번호 검사 / 로그인 제한 검사 / 2차 인증 검사 중 하나를 선택하는 화면 출력)
3. 데이터베이스 서버 화면 (접근 권한 검사 / 데이터 암호화 검사 / 백업 보안 검사 중 하나를 선택하는 화면 출력)

### 함수 설계
* web :  웹 서버의 보안 상태를 검사하는 함수
  * HTTPS 사용 여부, 입력값 검증, 파일 업로드 보안 항목을 선택하여 검사하고 취약점 발견 여부에 따라 위험 점수를 증가시킴
  * 1,2,3 외의 숫자를 입력했을 시 다시 입력하라는 문구를 출력함

  * 1,2,3 외의 숫자를 입력했을 시 다시 입력하라는 문구를 출력함

### 프로그램 사용 방식
1. 웹 서버 검사: 코드를 실행시키고 1을 입력한다 -> 1(HTTPS 사용 여부), 2(입력값 검증), 3(파일 업로드) 중 하나를 선택한다 -> 취약점이 발견되었다면 보안도 위험 점수가 올라갔는지 확인한다 >> rand() 함수를 사용했기 때문에 취약점이 있는 부분은 항상 바뀐다
2. 로그인 서버 검사: 코드를 실행시키고 2를 입력한다 -> 1(비밀번호), 2(로그인 시도 제한), 3(2차 인증) 중 하나를 선택한다 -> 취약점이 발견되었다면 보안도 위험 점수가 올라갔는지 확인한다 >> 마찬가지로 rand() 함수를 사용했기 때문에 취약점이 있는 부분은 항상 바뀐다
3. 데이터베이스 서버 검사: 코드를 실행시키고 3을 입력한다 -> 1(접근 권한), 2(데이터 암호화), 3(백업 보안) 중 하나를 선택한다 -> 취약점이 발견되었다면 보안도 위험 점수가 올라갔는지 확인한다 >> 마찬가지로 rand() 함수를 사용했기 때문에 취약점이 있는 부분은 항상 바뀐다
4. 탐색 종료: 모든 서버의 탐색이 종료되었다면 4를 입력한다 -> 프로그램이 종료되고 사용자가 취약점을 찾을 때마다 얻은 점수가 합산되어 보안도 위험 점수가 출력된다
